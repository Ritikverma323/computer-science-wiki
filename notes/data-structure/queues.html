Queue Data Structure

What is a Queue?

A Queue is a linear data structure that follows the FIFO (First In, First Out) principle. The first element inserted is the first to be removed.

Think of it like a line of people waiting at a ticket counter: the person who comes first is served first.

Features of Queues

  • Linear data structure with sequential order.
  • Follows FIFO (First In, First Out).
  • Insertion happens at the rear.
  • Deletion happens at the front.
  • Efficient for managing tasks in order of arrival.

Types of Queues

  • Simple Queue – Basic FIFO structure.
  • Circular Queue – Connects end to front to utilize space efficiently.
  • Priority Queue – Elements are dequeued based on priority, not order.
  • Deque (Double Ended Queue) – Insertion and deletion allowed from both ends.

Operations on Queues

  • Enqueue – Insert an element at the rear.
  • Dequeue – Remove an element from the front.
  • Peek/Front – Get the first element without removing it.
  • isEmpty() – Check if the queue is empty.
  • isFull() – Check if the queue is full (in case of fixed size).

Applications of Queues

  • Task scheduling in operating systems (CPU scheduling, disk scheduling).
  • Managing requests in web servers.
  • Printer task management (jobs handled in order).
  • Breadth-First Search (BFS) in graphs.
  • Handling asynchronous data (IO Buffers, Message Queues).
